Writing tests with maven: 

1. File location and naming

Test files go in: 
Backend/src/test/java/app/[service|model|repository]/[ClassName]Test.java

Tests for UserService.java should be in  
Backend/src/test/java/app/service/UserServiceTest.java
and tests for UserRepository.java should be in   
Backend/src/test/java/app/repository/UserRepositoryTest.java

2. Structure/Packaging/Imports

Also you have to make them part of the same subpackage as 
the class they test, so to test UserService.java you have to 
have a line at the top of the test file saying "package app.service;"
because it is a service.

ex: 

package app.[type];

import static org.junit.jupiter.api.Assertions.*;
-assertion utility methods to test assertions
import static org.mockito.Mockito.*;
import org.mockito.*;
-'mocking' framework for tests
import org.junit.jupiter.api.BeforeEach;
-helps reset state
import org.junit.jupiter.api.Test;
-lets methods be run as tests

public class YourClassTest {
    
    @Mock
    private DependencyClass dependency;  // Mock any dependencies
    
    @InjectMocks
    private YourClass yourClass;  // The class being tested
    
    @BeforeEach
    public void setUp() {
        MockitoAnnotations.openMocks(this);
    }
    
    @Test
    public void testSomething() {
        // set up test data
        when(dependency.someMethod()).thenReturn(expectedValue);
        
        // call the method
        Result result = yourClass.methodToTest();
        
        // verify the result
        assertEquals(expectedValue, result);
        verify(dependency, times(1)).someMethod();
    }
}

3. Run the test
be in the Backend directory
mvn test                      (run all tests)
mvn test -Dtest=UserServiceTest   (run one test)
